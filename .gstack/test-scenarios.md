# Wediary 테스트 시나리오

**앱:** wediary (Expo 54, React Native Web)
**URL:** http://localhost:8081
**테스트 방식:** 웹 브라우저 (Expo web dev mode)

---

## 사전 조건

- `cd app && npm run web` 으로 앱이 실행 중이어야 함
- 테스트 계정으로 로그인된 상태 기준 (미로그인 상태도 일부 포함)

---

## 시나리오 목록

### SC-01. 홈 화면 로드
**목적:** 앱 초기 진입 시 홈 화면이 정상 렌더링되는지 확인

1. `http://localhost:8081` 접속
2. 스크린샷: `sc01-home.png`
3. **확인 항목:**
   - "wediary" 로고(핑크) 표시
   - "설정" 버튼 우상단
   - 탭("예정" / "지난") 표시
   - 결혼식 목록 또는 빈 상태(💌) 표시
   - 우하단 "+" FAB 버튼
   - 콘솔 에러 없음

---

### SC-02. 탭 전환 (예정 ↔ 지난)
**목적:** 탭 전환 동작 확인

1. 홈 화면에서 "지난" 탭 클릭
2. 스크린샷: `sc02-tab-past.png`
3. **확인 항목:**
   - "지난" 탭이 활성화(흰 배경)
   - 지난 결혼식 목록 또는 빈 상태(📖) 표시
4. "예정" 탭 클릭해서 원복

---

### SC-03. 새 결혼식 추가 - 빈 폼
**목적:** 새 결혼식 화면 렌더링 확인

1. 홈에서 "+" 버튼 클릭
2. 스크린샷: `sc03-new-empty.png`
3. **확인 항목:**
   - 신랑/신부 이름 입력 필드
   - 날짜 선택기
   - 장소 입력 필드
   - 참석 여부 선택 (참석/불참/미정)
   - 청첩장 URL 입력 필드
   - "저장" 버튼

---

### SC-04. 새 결혼식 추가 - 폼 입력 후 저장
**목적:** 결혼식 생성 플로우 E2E 확인

1. "+" 버튼 클릭
2. 신랑: `테스트신랑`, 신부: `테스트신부` 입력
3. 날짜: 오늘 날짜 선택
4. 장소: `테스트홀` 입력
5. 참석 여부: "참석" 선택
6. 스크린샷: `sc04-new-filled.png`
7. "저장" 클릭
8. 스크린샷: `sc04-new-saved.png`
9. **확인 항목:**
   - 저장 후 홈으로 이동
   - 홈 목록에 새 카드 표시 (테스트신랑 ♥ 테스트신부)
   - D-Day 뱃지 표시 (오늘이면 D-Day, 미래면 D-N)
   - 참석 뱃지 "참석" (라임 배경)

---

### SC-05. 결혼식 상세 화면
**목적:** 상세 화면 렌더링 및 추억 기록 확인

1. 홈에서 결혼식 카드 클릭
2. 스크린샷: `sc05-detail.png`
3. **확인 항목:**
   - 헤더: 뒤로(←), 수정(✏️), 삭제(🗑️) 아이콘
   - 신랑♥신부 이름, 날짜, 장소, D-Day 표시
   - 참석 여부 뱃지 (클릭 시 변경 가능)
   - 메모 입력 필드
   - 축의금 입력 필드
   - "저장" 버튼

---

### SC-06. 추억 저장
**목적:** 메모/축의금 저장 동작 확인

1. 상세 화면에서:
   - 메모: `좋은 결혼식이었어요` 입력
   - 축의금: `50000` 입력
2. 스크린샷: `sc06-memory-filled.png`
3. "저장" 클릭
4. 스크린샷: `sc06-memory-saved.png`
5. **확인 항목:**
   - 저장 성공 피드백 표시 ("저장됨 ✓")
   - 화면 이탈 시 "저장하지 않은 변경사항" Alert 없음

---

### SC-07. 설정 화면
**목적:** 설정 화면 렌더링 확인

1. 홈에서 "설정" 클릭
2. 스크린샷: `sc07-settings.png`
3. **확인 항목:**
   - "← 뒤로" 버튼
   - 로그인한 사용자 이름/이메일 표시
   - "로그아웃" 버튼

---

### SC-08. 로그아웃 확인 모달
**목적:** 로그아웃 플로우 안전성 확인

1. 설정 → "로그아웃" 클릭
2. 스크린샷: `sc08-logout-modal.png`
3. **확인 항목:**
   - 확인 모달 표시
   - "취소" 클릭 시 모달 닫힘, 설정 유지
4. 취소 클릭 후 스크린샷: `sc08-logout-cancelled.png`

---

### SC-09. 개인정보처리방침
**목적:** 링크 및 화면 렌더링 확인

1. 설정 화면에서 개인정보처리방침 링크 클릭
2. 스크린샷: `sc09-privacy.png`
3. **확인 항목:**
   - 내용 정상 렌더링
   - "← 뒤로" 버튼 동작

---

### SC-11. 지난 결혼식 데이터 확인
**목적:** 이미 데이터가 입력된 지난 결혼식 카드 및 상세 화면 확인

1. "지난 결혼식" 탭 클릭
2. 스크린샷: `sc11-done-list.png`
3. **확인 항목 (목록):**
   - 카드에 신랑♥신부 이름, 날짜 표시
   - D-Day 뱃지 **없음** (지난 결혼식은 뱃지 미표시)
   - 참석 뱃지 표시 (참석/불참/미정)
   - 날짜 내림차순 정렬 (최근 결혼식이 맨 위)
4. 카드 클릭 → 상세 화면
5. 스크린샷: `sc11-done-detail.png`
6. **확인 항목 (상세):**
   - 날짜 표시가 `N일 전` 형식으로 표시
   - 저장된 메모, 감정 태그, 축의금 값이 그대로 표시
   - 청첩장 링크(invite_url)가 있으면 링크 아이콘과 함께 표시
   - 수정(✏️) 클릭 시 → 수정 폼에 청첩장 링크 포함 모든 필드 채워져 있음

---

### SC-10. 비로그인 상태 (선택)
**목적:** 미인증 사용자 경험 확인

1. 로그아웃 후 앱 접속
2. 스크린샷: `sc10-login.png`
3. **확인 항목:**
   - 로그인 화면 렌더링
   - 소셜 로그인 버튼들 표시
   - "로그인 없이 계속 (테스트)" 버튼이 **개발 모드에서만** 표시
