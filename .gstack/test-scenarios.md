# Wediary 테스트 시나리오

**앱:** wediary (Expo 54, React Native)

---

## 플랫폼별 실행 방법

| 플랫폼 | 실행 명령 | 비고 |
|--------|-----------|------|
| **Web** | `cd app && npm run web` → `http://localhost:8081` | 빠른 UI 확인용. 카메라/캘린더 불가 |
| **iOS (Xcode)** | `cd app && npx expo run:ios` | 시뮬레이터. 카카오 로그인 + 전 기능 가능 |
| **Android** | `cd app && npx expo run:android` | 에뮬레이터. 카카오 로그인 + 전 기능 가능 |

### 플랫폼별 기능 지원 범위

| 기능 | Web | iOS 시뮬레이터 | Android 에뮬레이터 |
|------|-----|---------------|-------------------|
| 카카오 로그인 | ❌ (테스트 익명 로그인) | ✅ | ✅ |
| 결혼식 CRUD | ✅ | ✅ | ✅ |
| 캘린더 추가 | ❌ | ✅ | ✅ |
| OCR 스캔 (카메라) | ❌ | ❌ (시뮬레이터 카메라 없음) | ❌ |
| OCR 스캔 (갤러리) | ❌ | ✅ | ✅ |
| 청첩장 URL 파싱 | ✅ | ✅ | ✅ |

- iOS/Android는 카카오 로그인 사용

## SC-01. 홈 화면 로드
**목적:** 앱 초기 진입 시 홈 화면이 정상 렌더링되는지 확인
**플랫폼:** Web / iOS / Android

1. 앱 실행 후 로그인
2. 스크린샷: `sc01-home.png`
3. **확인 항목:**
   - "wediary" 로고(핑크) 표시
   - "설정" 버튼 우상단
   - 탭("예정" / "지난 결혼식") 표시
   - 결혼식 목록 또는 빈 상태(💌) 표시
   - 우하단 "+" FAB 버튼
   - 콘솔 에러 없음
4. **에러 상태 확인 (네트워크 차단 후):**
   - 에러 메시지 + "다시 시도" 버튼 표시
   - "다시 시도" 탭 시 재조회 시도

## SC-02. 탭 전환 (예정 ↔ 지난)
**목적:** 탭 전환 동작 확인
**플랫폼:** Web / iOS / Android

1. 홈 화면에서 "지난 결혼식" 탭 클릭
2. 스크린샷: `sc02-tab-past.png`
3. **확인 항목:**
   - "지난 결혼식" 탭이 활성화(핑크 배경)
   - 지난 결혼식 목록 또는 빈 상태(📖) 표시
4. "예정" 탭 클릭해서 원복

---

## SC-03. 새 결혼식 추가 - 빈 폼
**목적:** 새 결혼식 화면 렌더링 확인
**플랫폼:** Web / iOS / Android

1. 홈에서 "+" 버튼 클릭
2. 스크린샷: `sc03-new-empty.png`
3. **확인 항목:**
   - 청첩장 URL 입력 필드 + "불러오기" 버튼
   - 카메라 촬영 / 갤러리 선택 버튼
   - 신랑/신부 이름 입력 필드
   - 날짜 선택기 (오늘 날짜 기본값)
   - 장소 입력 필드
   - 참석 여부 선택 (참석/불참/미정, 기본값 미정)
   - 헤더: 취소 / 저장 버튼
4. **엣지케이스 확인:**
   - 신랑/신부 이름 없이 "저장" → 에러 배너 표시 ("신랑 또는 신부 이름을 입력해주세요")
   - 이름 입력 후 "취소" → "저장하지 않은 변경사항" Alert 표시
   - `http://` 없는 URL 입력 후 "불러오기" → 형식 오류 Alert 표시

---

## SC-04. 새 결혼식 추가 - 수동 입력 후 저장
**목적:** 결혼식 생성 플로우 E2E 확인
**플랫폼:** Web / iOS / Android

1. "+" 버튼 클릭
2. 신랑: `테스트신랑`, 신부: `테스트신부` 입력
3. 날짜: 미래 날짜 선택 (예: 한 달 뒤)
4. 장소: `테스트홀` 입력
5. 참석 여부: "참석" 선택
6. 스크린샷: `sc04-new-filled.png`
7. "저장" 클릭
8. 스크린샷: `sc04-new-saved.png`
9. **확인 항목:**
   - 저장 후 홈으로 이동
   - 홈 "예정" 탭에 새 카드 표시 (테스트신랑 ♥ 테스트신부)
   - D-Day 뱃지 표시 (D-N, 날짜 무관 모든 예정 결혼식에 표시)
   - 참석 뱃지 "참석" (라임 배경)

---

## SC-05. 결혼식 상세 화면
**목적:** 상세 화면 렌더링 및 추억 기록 확인
**플랫폼:** Web / iOS / Android

1. 홈에서 결혼식 카드 클릭
2. 스크린샷: `sc05-detail.png`
3. **확인 항목:**
   - 헤더: 뒤로(`<`), 수정(✏️), 삭제(🗑️) 아이콘
   - 신랑♥신부 이름(가에구 폰트), 날짜, 장소, D-Day 표시
   - 장소 없는 결혼식: 장소 행 자체가 표시되지 않아야 함 (복사 버튼 포함)
   - 참석 여부 뱃지 → 탭 시 인라인 3-버튼 선택기(참석/불참/미정) 전환, 선택 즉시 저장
   - invite_url 있는 결혼식: URL 행 탭 시 브라우저 열림, 길게 누르면 클립보드 복사
   - 예정 결혼식: "캘린더에 추가" 버튼 표시 / 지난 결혼식: 캘린더 버튼 없음
   - 메모 입력 필드
   - 축의금 빠른 선택 (5만/10만/15만/20만) + 직접 입력
   - "저장" 버튼

## SC-06. 추억 저장
**목적:** 메모/축의금 저장 동작 확인
**플랫폼:** Web / iOS / Android

1. 상세 화면에서:
   - 메모: `좋은 결혼식이었어요` 입력
   - 축의금: `50000` 직접 입력 또는 "5만" 탭
2. 스크린샷: `sc06-memory-filled.png`
3. "기억 저장" 클릭
4. 스크린샷: `sc06-memory-saved.png`
5. **확인 항목:**
   - 저장 버튼이 "저장됨 ✓" (라임색)으로 2초간 변경
   - 다시 들어오면 메모/축의금 값 유지
   - 미저장 상태에서 뒤로 가면 "저장하지 않은 변경사항" Alert 표시

## SC-11. 지난 결혼식 데이터 확인
**목적:** 기존 데이터가 있는 지난 결혼식 목록/상세 확인
**플랫폼:** Web / iOS / Android

1. "지난 결혼식" 탭 클릭
2. 스크린샷: `sc11-done-list.png`
3. **확인 항목 (목록):**
   - 카드에 신랑♥신부 이름, 날짜 표시
   - D-Day 뱃지 **없음** (지난 결혼식)
   - 날짜 내림차순 정렬 (최근이 맨 위)
4. 카드 클릭 → 상세 화면
5. 스크린샷: `sc11-done-detail.png`
6. **확인 항목 (상세):**
   - 날짜가 `N일 전` 형식 표시
   - 저장된 메모/축의금 값 표시
   - invite_url 있으면 링크 아이콘과 함께 표시
   - 수정(✏️) 클릭 시 → 수정 폼에 모든 필드(invite_url 포함) 채워져 있음

## SC-12. 모바일 청첩장 URL 파싱
**목적:** 청첩장 링크 → 자동 폼 입력 동작 확인
**플랫폼:** Web / iOS / Android (네트워크 필요)

1. "+" 버튼 → 새 결혼식 화면
2. 청첩장 URL 필드에 실제 모바일 청첩장 URL 붙여넣기
   - 예: 카카오 청첩장, 웨딩21, 더카드 등
3. "불러오기" 버튼 탭
4. 스크린샷: `sc12-parse-loading.png` (로딩 중)
5. 스크린샷: `sc12-parse-result.png` (결과)
6. **확인 항목:**
   - 로딩 중 버튼에 스피너 표시
   - 파싱 성공 시: 신랑/신부/날짜/장소 필드 자동 입력
   - 파싱 실패 시: "파싱 실패" Alert 표시 (앱 크래시 없음)
   - invite_url 필드에 입력한 URL이 그대로 유지됨

## SC-13. OCR — 갤러리 청첩장 스캔
**목적:** 갤러리 이미지 OCR → 자동 폼 입력 동작 확인
**플랫폼:** iOS 시뮬레이터 / Android 에뮬레이터 (Web 불가)

1. "+" 버튼 → 새 결혼식 화면
2. "갤러리 선택" 버튼 탭
3. 청첩장 이미지 선택 (사전에 시뮬레이터 사진첩에 저장 필요)
4. 스크린샷: `sc13-ocr-result.png`
5. **확인 항목:**
   - 갤러리 권한 요청 → 허용
   - OCR 처리 중 버튼에 로딩 표시
   - 인식 성공 시: 신랑/신부/날짜/장소 중 하나 이상 자동 입력
   - 인식 실패 시: "인식 실패" Alert 표시 (앱 크래시 없음)

> **시뮬레이터 사진 추가 방법:** 시뮬레이터에서 Safari → 이미지 길게 탭 → 사진에 저장

## SC-14. 캘린더 자동 추가
**목적:** 결혼식 일정을 기기 캘린더에 등록하는 동작 확인
**플랫폼:** iOS 시뮬레이터 / Android 에뮬레이터 (Web 불가)

1. 결혼식 상세 화면 진입
2. "캘린더" 버튼 탭
3. 스크린샷: `sc14-calendar-permission.png` (권한 요청 시)
4. 스크린샷: `sc14-calendar-added.png` (추가 후)
5. **확인 항목:**
   - 최초 실행 시 캘린더 접근 권한 요청
   - 권한 허용 후 "추가됨 ✓" 피드백 (라임색, 2초)
   - 기기 캘린더 앱에서 이벤트 확인 (신랑♥신부, 날짜, 장소)
   - 권한 거부 시 적절한 에러 메시지
